### 10. Ingestion Worker (Celery)

Build in `ingestion-worker/`.  
**Dockerfile** based on `python:3.11-slim`.  
**`tasks.py`** defines Celery app (broker `redis://redis:6379/1`) and these tasks:

1. **`ingest_document(document_id, file_path, department, file_type)`**  
   - Retries 3× with exponential backoff (60s, 120s, 240s).  
   - Timeout: 30 min.  
   - **Steps**:  
     1. Update document status to `processing` in PostgreSQL.  
     2. If file already exists (matched by filename+department), delete old Qdrant vectors + BM25 entries + update version.  
     3. Parse file with **Unstructured.io** (`partitioner.auto`). For scanned PDFs, use `pdf2image` + `pytesseract`.  
     4. Chunk text according to document type:  
        - SOPs/policies: 512 tokens, 50 overlap (RecursiveCharacterTextSplitter)  
        - Manuals: 768 tokens, 100 overlap  
        - Excel: 256 tokens, no overlap (row‑wise)  
        - Notes: 384 tokens, 64 overlap  
     5. For each chunk, generate embedding using Ollama (`nomic-embed-text`, 768‑dim).  
     6. Upload chunks to Qdrant collection `documents` with payload: `{text, filename, page, section, document_id, department, file_type, uploaded_at, version}`.  
     7. Build/update BM25 in‑memory index (using `rank_bm25`) and persist to Redis DB 3 or a file; the query‑engine will load it.  
     8. Upload original file to MinIO bucket `documents` under `/{department}/{filename}`.  
     9. Update document status to `indexed`, set chunk_count, indexed_at.  
     10. On failure: status `failed`, write error to `ingestion_jobs`, send alert (webhook/log).  
2. **`delete_document_vectors(document_id)`** – delete all Qdrant points with matching `document_id`.  
3. **`run_ragas_evaluation()`** – weekly, sample 50 answered queries with feedback, compute faithfulness, answer relevancy, context precision, recall, store results.  
4. **`backup_postgres()`** – nightly at 03:00, `pg_dump` gzip → MinIO `backups/postgres/YYYY-MM-DD.sql.gz`, retain 30 days.  
5. **`backup_qdrant()`** – weekly Sun 03:30, snapshot via Qdrant API → MinIO `backups/qdrant/`.

**Celery Beat schedule** (in `tasks.py`):  
- `backup-postgres-nightly` at `crontab(hour=3, minute=0)`  
- `backup-qdrant-weekly` at `crontab(hour=3, minute=30, day_of_week=0)`  
- `ragas-evaluation-weekly` at `crontab(hour=2, minute=0, day_of_week=0)`

**Worker** (`worker.py`): just starts Celery worker listening on queue `ingestion`.

### 11. Query Engine

Build in `query-engine/`.  
**Dockerfile** as a FastAPI service (port 8001).  
**`engine.py`** provides `/health` and `/query` (internal API).  
**Query pipeline:**

1. Receive `question`, optional `filters` (department, file_type, date_range, document name, version).
2. Embed question with `nomic-embed-text` → 768‑d vector.
3. **Hybrid search**:  
   - Qdrant vector search: top‑20, with optional payload filters.  
   - BM25 keyword search: top‑20 using in‑memory index (loaded from Redis/file at startup).  
   - **Reciprocal Rank Fusion**: score = 1/(k+rank), k=60, merge & deduplicate → top‑40.  
4. **Reranker**: download `cross-encoder/ms-marco-MiniLM-L-6-v2` on startup, score all 40 (query, chunk) pairs, keep top‑5.  
5. **Prompt assembly**: use the exact template below.
6. **LLM generation**: call Ollama Llama 3 8B (streaming if possible), concatenate answer.
7. Return answer + `sources` list (filename, page, section, score).
8. Log full trace to Langfuse (spans for each step). Emit Prometheus metrics (query counter, latency histogram, etc.).
9. Cache identical questions using Redis (hashing question + filters) to skip re‑computation.

**Prompt Template:**
```
SYSTEM:
You are a precise assistant for engineering, operations, and maintenance teams.
Rules:
1. Answer ONLY using the provided context. Never use external knowledge.
2. Always cite: [Source: {filename}, Page {page}, Section: {section}]
3. If the answer is not in the context, respond exactly:
   "This information was not found in the available documents.
    Recommended documents to check manually: {suggest_category}"
4. For safety-critical procedures, add: WARNING: Always verify with
   the original document before performing this procedure.
5. Be concise. Use numbered steps for procedures. Use bullet points for lists.

CONTEXT:
[Chunk 1 - Score: 0.94]: {text} [Source: {filename}, Page {page}]
[Chunk 2 - Score: 0.91]: {text} [Source: {filename}, Page {page}]
...

QUESTION: {user_question}

ANSWER:
```
